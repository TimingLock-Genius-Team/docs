# Eulr — 产品设计文档

> **XLayer 上基于 sat1 曲线机制的无许可代币发射台**

---

## 1. 产品定位

Eulr 是一个运行在 **XLayer 主网**的无许可代币发射台，任何人可以一键部署使用 **sat1 Bonding Curve** 机制的 ERC-20 代币。

### 核心特点

- **定位**：pump.fun 的 XLayer 增强版
- **去中心化**：不持有任何资金，不设后门
- **开源透明**：代码全部开源，链上可验证

---

## 2. 核心机制（继承自 sat1）

### 2.0 ⭐ 核心亮点：动态销毁指数（基于 Uniswap Hook V4）

Eulr 的核心创新在于利用 **Uniswap Hook V4** 将代币价格与销毁指数动态关联，实现「**跌难涨易**」的代币经济模型，从而更好地构建社区共识。

#### 机制原理

```
销毁指数 = f(当前价格)

当价格 ↑ 时 → 销毁指数 ↓（卖出时销毁比例降低）
当价格 ↓ 时 → 销毁指数 ↑（卖出时销毁比例升高）
```

#### 设计目标

| 目标 | 实现方式 |
|------|----------|
| **跌难** | 价格下跌时，卖出触发更高的销毁比例，增加抛售成本，抑制恐慌性抛盘 |
| **涨易** | 价格上涨时，卖出销毁比例降低，降低获利了结成本，鼓励持有与买入 |
| **共识构建** | 通过经济激励机制，让持有者与项目利益深度绑定 |

#### 技术实现

通过 Uniswap V4 的 Hook 机制，在每笔交易的 `beforeSwap` / `afterSwap` 阶段注入自定义逻辑：

1. **价格读取**：从 Pool 状态获取当前 `sqrtPriceX96`
2. **指数计算**：根据价格偏离基准值的程度，动态计算销毁指数
3. **代币销毁**：在卖出操作中，按销毁指数比例将部分代币发送至黑洞地址

#### 示例场景

| 场景 | 价格变化 | 销毁指数 | 效果 |
|------|----------|----------|------|
| 市场恐慌 | -30% | 高（如 5%） | 卖出 100 代币，5 枚被销毁，实际到手 95 枚价值的 OKB |
| 价格稳定 | ±5% | 中（如 1%） | 正常交易摩擦 |
| 市场狂热 | +50% | 低（如 0.1%） | 几乎无额外销毁，鼓励流动性 |

#### 优势总结

- **反脆弱性**：下跌时自动收缩流通量，形成价格支撑
- **正向飞轮**：上涨时降低摩擦，加速价格发现
- **链上透明**：所有销毁记录可追溯，销毁指数算法开源
- **无需信任**：完全由智能合约自动执行，无人工干预

### 2.1 Bonding Curve 公式

```
price(okb)  = (S/K) × e^(okb/S)      // 边际价格
minted(okb) = K × (1 - e^(-okb/S))   // 累积铸造量
```

**曲线参数（与 sat1 一致）：**

| 参数 | 值 | 说明 |
|------|-----|------|
| `K` | 21,000,000 | 最大供应量上限 |
| `S` | 100 OKB | 曲线形状参数 |

### 2.2 唯一状态变量

合约只存储 `okbCum`（累计 OKB 投入量），所有派生值均从此单一变量计算：

| 派生值 | 计算方式 |
|--------|----------|
| 当前价格 | `Curve.marginalPrice(okbCum)` |
| 已铸造量 | `Curve.totalMinted(okbCum)` |
| 卖出报价 | `burnFor(Curve.totalMinted(okbCum), share)` |
| 是否毕业 | `Curve.totalMinted(okbCum) >= 99% × K` |

### 2.3 费用机制

| 操作 | 费率 | 去向 |
|------|------|------|
| **MINT** | 0.3% | 团队多签钱包 |
| **BURN** | 0.3% | 团队多签钱包 |

**毕业后处理**：自动迁移至 Uniswap V4 添加流动性池，LP Token 直接燃烧

### 2.4 防机器人机制

- **单次买入上限**：10 OKB
- **同块限制**：同一区块内买入后不可卖出（防止同块套利）
- **熵值机制**：早期启动窗口可引入 entropy 对抗精确区块自动化

### 2.5 毕业机制（selfDeprecated）

当 `totalMinted >= 99% × K` 时触发：

1. ✅ 买入永久停止
2. ✅ 持有者仍可通过 Hook 卖出（曲线反向运行）
3. ✅ 代币继续存在，可在任意二级市场交易

## 3. 产品功能模块

### 3.1 发现页（Explore）

| 功能 | 描述 |
|------|------|
| **代币列表** | 按 新上线 / 热度 / 临近毕业 筛选 |
| **代币卡片** | 显示名称、代码、曲线进度、当前价格、储备量、24h 交易量 |
| **搜索** | 按代币名称、合约地址搜索 |
| **实时更新** | 通过链上事件订阅实时刷新进度条 |

### 3.2 代币详情页（Token Detail）

#### 信息展示区
- 代币名称、符号、合约地址（可复制）
- 创建时间、创建者地址
- 社交链接（Twitter、Telegram、网站）
- 曲线进度条（0% → 99%），标注关键里程碑

#### 价格图表
- **时间维度**：1m / 5m / 1h / 1d
- **双线展示**：边际价格曲线 + 实际成交均价

#### 买卖面板
- **买入**：输入 OKB 数量 → 显示预计获得代币（扣除 0.3% 费用后）
- **卖出**：输入代币数量 → 显示预计获得 OKB（扣除 0.3% 费用后）
- **显示信息**：当前价格、价格影响（price impact）、最大买入限额（10 OKB）
- **警告**：`selfDeprecated` 触发时禁用买入，显示「铸造已结束」

#### 链上统计
- 当前 `okbCum`、已铸造量（%）、储备 OKB、持有人数

#### 交易历史
- 链上事件流：时间、类型（买/卖）、OKB 数量、代币数量、地址

### 3.3 创建代币页（Create）

#### Step 1 — 基本信息
| 字段 | 限制 |
|------|------|
| 代币名称（Name） | 最多 32 字符 |
| 代币代码（Symbol） | 最多 8 字符，自动转大写 |
| 描述（Description） | 最多 280 字符 |
| 图片上传 | PNG / JPG，建议 512×512 |

#### Step 2 — 社交信息（可选）
- Twitter / X 链接
- Telegram 链接
- 官网链接

#### Step 3 — 曲线预览
- 展示 sat1 曲线（固定参数，不可修改）
- 关键价格点：0 / 10 / 50 / 100 OKB 时的边际价格
- 毕业所需约 100 OKB 净投入

#### Step 4 — 部署
- 调用 sat1 Hook Deployer 工厂合约
- Gas 费由创建者承担，**Eulr 不收额外部署费**
- 部署成功后自动跳转到代币详情页

### 3.4 投资组合页（Portfolio）

- **当前持仓**：代币、数量、当前价值（按曲线卖出报价计算）
- **历史交易**：买入/卖出记录
- **未实现盈亏**：基于加权平均成本

## 5. 关键 UX 原则

### 5.1 透明优先

- 所有关键数字（储备、`okbCum`、费用去向）在链上可验证，界面直接展示
- 费用面板明确标注「0.3% 打入团队多签」
- 价格影响警告：**>5% 橙色**，**>15% 红色**

### 5.2 机制教育

- 创建页和详情页均有曲线原理简介（可折叠）
- 毕业条件在进度条旁明确说明
- 新用户首次访问展示「Eulr 与普通 Meme Launchpad 的区别」弹窗

### 5.3 无许可承诺

- 界面不可屏蔽任何已部署代币（无管理员删除功能）
- 合约地址页面永久可访问，即使前端下线
- 合约代码链接（Verified）在每个代币页显眼处展示
